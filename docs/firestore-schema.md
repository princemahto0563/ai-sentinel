# Firestore Schema — AI Sentinel

## Collection: `detections`

Each document ID is a UUID generated by the Cloud Function.

```json
{
  "id": "3f2504e0-4f89-11d3-9a0c-0305e82c3301",
  "image_url": "gs://YOUR_PROJECT_ID.appspot.com/detections/3f2504e0.jpg",
  "source": "CCTV",
  "track_segment": "42A",
  "label": "Crack",
  "raw_label": "Crack",
  "confidence": 0.92,
  "bounding_boxes": [
    {
      "label": "Crack",
      "confidence": 0.92,
      "x": 0.12,
      "y": 0.24,
      "width": 0.44,
      "height": 0.08
    }
  ],
  "location": {
    "lat": 28.6139,
    "lng": 77.2090
  },
  "severity": "Critical",
  "suggested_action": "Emergency Brake Alert. Halt all train movement on Segment 42A. Dispatch track engineering crew immediately.",
  "technical_report": "Critical track fracture detected at Segment 42A with 92% confidence. Structural integrity compromised. Immediate intervention required before resuming operations.",
  "decision_model": "gemini-1.5-flash",
  "lives_saved_estimate": 1,
  "metadata": {
    "camera_id": "cctv-ndls-042a",
    "train_id": "EXP-12951",
    "corridor": "Delhi-Mumbai",
    "submitted_from": "firebase_hosting_dashboard"
  },
  "timestamp": "<Firestore serverTimestamp>"
}
```

## Field Reference

| Field | Type | Values | Description |
|---|---|---|---|
| `id` | `string` | UUID | Document ID |
| `image_url` | `string` | `gs://…` | Cloud Storage path |
| `source` | `string` | CCTV, locomotive_camera, drone, uploaded_image | Image origin |
| `track_segment` | `string` | e.g. "42A", "NDLS-BRC" | Railway segment identifier |
| `label` | `string` | Crack, Missing Fishplate, Obstacle, Normal | Primary Vertex AI label |
| `raw_label` | `string` | same as label | Unmodified Vertex AI output |
| `confidence` | `number` | 0.0–1.0 | Vertex AI confidence score |
| `bounding_boxes` | `array` | see below | Detected object positions |
| `location.lat` | `number` | –90 to 90 | GPS latitude |
| `location.lng` | `number` | –180 to 180 | GPS longitude |
| `severity` | `string` | Critical, Medium, Low | Gemini severity classification |
| `suggested_action` | `string` | — | Gemini operational directive |
| `technical_report` | `string` | — | Gemini technical narrative |
| `decision_model` | `string` | gemini-1.5-flash, rule-fallback | Model used for classification |
| `lives_saved_estimate` | `number` | 0 or 1 | 1 for Critical detections |
| `metadata` | `map` | — | Camera ID, train ID, corridor |
| `timestamp` | `timestamp` | Firestore Timestamp | Server-side write time |

## Bounding Box Schema

```json
{
  "label": "Crack",
  "confidence": 0.92,
  "x": 0.12,
  "y": 0.24,
  "width": 0.44,
  "height": 0.08
}
```

All values normalised 0.0–1.0 relative to image dimensions.

## Derived Analytics (computed, not stored)

```json

  "track_segment": "42A",
  "detections": 4,
  "critical_events": 2,
  "medium_events": 1,
```

## Firestore Indexes

Two composite indexes are required (defined in `firestore.indexes.json`):

1. `severity ASC` + `timestamp DESC` — for severity-filtered history queries
2. `track_segment ASC` + `timestamp DESC` — for segment-specific queries
